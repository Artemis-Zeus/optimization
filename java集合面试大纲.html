🔧 重要概念和术语

核心接口

  • Collection ：集合框架的根接口

  • List ：有序可重复集合接口

  • Set ：无序不重复集合接口

  • Map ：键值对映射接口

  • Queue ：队列接口

  • Deque ：双端队列接口

实现类术语

  • ArrayList ：动态数组实现的List

  • LinkedList ：双向链表实现的List和Deque

  • HashMap ：哈希表实现的Map

  • TreeMap ：红黑树实现的有序Map

  • HashSet ：基于HashMap实现的Set

  • PriorityQueue ：基于堆实现的优先队列

算法相关术语

  • Hash Collision（哈希冲突） ：不同key计算出相同hash值

  • Load Factor（负载因子） ：哈希表中元素数量与容量的比值

  • Resize（扩容） ：动态调整底层数组大小

  • Red-Black Tree（红黑树） ：自平衡的二叉搜索树

  • Segment（分段） ：ConcurrentHashMap的分段锁设计

性能相关术语

  • Time Complexity（时间复杂度） ：算法执行时间与数据规模的关系

  • Space Complexity（空间复杂度） ：算法空间占用与数据规模的关系

  • Amortized Analysis（摊还分析） ：平均情况下的复杂度分析

  • Capacity（容量） ：集合底层数组的大小

  • Size（大小） ：集合中实际元素的数量

🌟 学习重点提示

核心原理理解

  1. 哈希表原理 - HashMap的核心实现机制

  2. 红黑树特性 - 理解自平衡二叉搜索树的优势

  3. 动态扩容 - ArrayList和HashMap的扩容策略

  4. 并发安全 - ConcurrentHashMap的线程安全实现

常见误区避免

  1. 盲目使用Vector - 现代应用很少需要Vector的同步特性

  2. 忽视初始容量 - 合理设置初始容量可以避免多次扩容

  3. 错误实现equals/hashCode - 影响HashSet和HashMap的正确性

  4. 在遍历时修改集合 - 可能导致ConcurrentModificationException
