深度优先搜索详解

🎯 核心思想

深度优先搜索(Depth-First Search, DFS)是一种用于遍历或搜索树或图的算法。DFS会尽可能深地搜索一个分支，直到该分支的末端，然后回溯到上一个节点，继续搜索其他分支。

基本原理

核心特征：

  • 🔹 深度优先 ：优先访问更深层的节点

  • 🔹 回溯机制 ：当无路可走时返回上一层继续搜索

  • 🔹 栈结构 ：可用递归（系统栈）或显式栈实现

  • 🔹 记忆访问状态 ：避免重复访问同一节点

🧠 算法思维模式

DFS实现方式

1. 递归实现（最常用）

特征： 利用函数调用栈自动管理回溯

dfs(当前节点): if 当前节点已访问 or 不符合条件: return 标记当前节点为已访问 处理当前节点 for 每个邻居节点: dfs(邻居节点) // 可选：撤销标记（用于回溯）

2. 迭代实现（显式栈）

特征： 手动管理栈，更直观地看到搜索过程

stack = [起始节点] visited = set() while stack: current = stack.pop() if current in visited: continue visited.add(current) 处理当前节点 for 邻居 in current的邻居们: if 邻居 not in visited: stack.push(邻居)

DFS应用场景分类

1. 路径搜索

  • 找到从起点到终点的路径

  • 计算路径数量

  • 寻找最短/最长路径

2. 连通性检测

  • 判断图是否连通

  • 找到连通分量

  • 检测环的存在

3. 拓扑排序

  • 检测有向图中的环

  • 按依赖关系排序

4. 回溯算法

  • 生成所有可能的解

  • 约束满足问题

  • 组合优化问题

📊 经典题目分析

题目1：岛屿数量

问题描述： 给你一个由'1'（陆地）和'0'（水）组成的的二维网格，请你计算网格中岛屿的数量。岛屿总是被水包围，并且每座岛屿只能由水平方向和/或竖直方向上相邻的陆地连接形成。

DFS策略： 遇到'1'时开始DFS，将整个岛屿标记为已访问

图解演示

解题思路：

  1. 遍历整个网格

  2. 遇到'1'时，计数器+1，启动DFS

  3. DFS将当前岛屿的所有'1'标记为'0'

  4. 继续遍历，寻找下一个岛屿

题目2：路径总数

问题描述： 给定一个m×n的网格，机器人从左上角(0,0)开始，想要到达右下角(m-1,n-1)。机器人每次只能向下或向右移动一步。请问总共有多少条不同的路径？

DFS策略： 从起点开始，探索所有可能的路径

图解演示

解题思路：

  1. 递归定义：dfs(i,j) = 从位置(i,j)到终点的路径数

  2. 递归关系：dfs(i,j) = dfs(i+1,j) + dfs(i,j+1)

  3. 边界条件：到达终点返回1，越界返回0

  4. 优化：使用记忆化避免重复计算

题目3：N皇后问题

问题描述： n皇后问题研究的是如何将n个皇后放置在n×n的棋盘上，并且使皇后彼此之间不能相互攻击。

DFS策略： 逐行放置皇后，用回溯处理冲突

图解演示

解题思路：

  1. 逐行放置皇后，每行只能放一个

  2. 对于每一行，尝试每一列

  3. 检查是否与之前的皇后冲突

  4. 如果不冲突，递归搜索下一行

  5. 如果冲突或无解，回溯到上一行

题目4：单词搜索

问题描述： 给定一个二维网格和一个单词，找出该单词是否存在于网格中。单词必须按照字母顺序，通过相邻的单元格内的字母构成。

DFS策略： 从每个可能的起点开始DFS搜索单词

图解演示

解题思路：

  1. 遍历网格，找到与单词首字母匹配的位置

  2. 从该位置开始DFS搜索

  3. 在DFS中检查当前字符是否匹配

  4. 标记访问状态，防止重复使用同一位置

  5. 递归搜索四个方向，回溯时恢复状态

🔧 DFS设计技巧

状态管理策略

1. 访问标记

// 全局标记（适用于无需恢复的情况） visited[node] = true // 临时标记（适用于回溯的情况） visited[node] = true dfs(next_node) visited[node] = false // 回溯时恢复

2. 路径记录

path = [] path.append(current_node) dfs(next_node) path.pop() // 回溯时移除

3. 状态传递

// 通过参数传递状态 dfs(node, path, visited, target) // 通过返回值传递结果 result = dfs(node)

剪枝技巧

1. 可行性剪枝

if 当前状态不可能达到目标: return // 提前终止

2. 最优性剪枝

if 当前解已经不如已知最优解: return // 提前终止

3. 重复状态剪枝

if 当前状态已经访问过: return // 避免重复搜索

📈 复杂度分析

时间复杂度

  • 树结构DFS ：O(V)，V为节点数

  • 图结构DFS ：O(V + E)，E为边数

  • 网格DFS ：O(m × n)，m和n为网格尺寸

  • 回溯算法 ：O(b^d)，b为分支因子，d为深度

空间复杂度

  • 递归调用栈 ：O(h)，h为最大深度

  • 访问标记 ：O(V)，V为节点数

  • 路径存储 ：O(h)，h为路径长度

✅ 适用场景识别

适合DFS的问题特征

  1. 🌳 树或图的遍历

    • 需要访问所有节点

    • 寻找特定路径或节点

  2. 🔍 路径搜索问题

    • 寻找从起点到终点的路径

    • 计算路径数量或最优路径

  3. 🔄 回溯算法问题

    • 生成所有可能的组合或排列

    • 约束满足问题

  4. 🏗️ 拓扑结构分析

    • 检测环的存在

    • 连通性分析

识别关键词

✅ "遍历所有可能" ✅ "寻找路径" ✅ "连通性/连通分量" ✅ "回溯/撤销操作" ✅ "深度探索"

不适合DFS的情况

❌ 需要最短路径 （BFS更适合） ❌ 内存限制严格 （可能栈溢出） ❌ 只需要浅层搜索 （BFS更高效） ❌ 需要层次遍历

💡 解题模板

递归DFS模板

function dfs(当前节点, 其他参数): // 1. 终止条件 if 到达终点 or 不满足条件: 处理结果 return // 2. 标记访问状态 mark visited[当前节点] = true // 3. 处理当前节点 处理当前节点的逻辑 // 4. 递归搜索邻居 for each 邻居 in 当前节点的邻居: if 邻居未访问 and 满足条件: dfs(邻居, 更新后的参数) // 5. 回溯（如果需要） 撤销当前节点的操作 mark visited[当前节点] = false

迭代DFS模板

function dfs_iterative(起始节点): stack = [起始节点] visited = set() while stack不为空: current = stack.pop() if current已访问: continue visited.add(current) 处理当前节点 for each 邻居 in current的邻居: if 邻居未访问: stack.push(邻居) return 结果

回溯DFS模板

function backtrack(路径, 选择列表): // 终止条件 if 满足结束条件: 结果.add(路径的副本) return for choice in 选择列表: // 做选择 路径.add(choice) // 递归 backtrack(路径, 新的选择列表) // 撤销选择 路径.remove(choice)
