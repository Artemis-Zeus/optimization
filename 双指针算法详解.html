双指针算法详解

🎯 核心思想

双指针算法是一种使用两个指针在数据结构中进行遍历和操作的技术。通过巧妙地移动两个指针，可以在线性时间内解决许多原本需要暴力枚举的问题，将时间复杂度从O(n²)降低到O(n)。

基本原理

核心特征：

  • 🔹 两个指针协同工作 ：通过两个指针的相对位置关系求解问题

  • 🔹 降低时间复杂度 ：避免嵌套循环，从O(n²)降到O(n)

  • 🔹 空间效率高 ：通常只需要O(1)的额外空间

  • 🔹 思路简洁清晰 ：逻辑直观，易于理解和实现

🧠 算法思维模式

双指针类型分类

1. 对撞指针（相向而行）

特征： 两个指针分别从数组两端开始，向中间移动

left = 0, right = n - 1 left向右移动，right向左移动 直到left >= right

2. 快慢指针（同向而行）

特征： 两个指针从同一位置开始，以不同速度移动

slow = 0, fast = 0 fast移动速度通常是slow的2倍 用于链表环检测、寻找中点等

3. 滑动窗口指针

特征： 两个指针维护一个窗口，根据条件调整窗口大小

left = 0, right = 0 right不断向右扩展 left根据条件向右收缩

4. 分离双指针

特征： 两个指针分别遍历不同的数据结构

指针1遍历数组A 指针2遍历数组B 根据比较结果移动相应指针

📊 经典题目分析

题目1：两数之和II（有序数组）

问题描述： 给定一个已按升序排列的整数数组numbers，请你从数组中找出两个数满足相加之和等于目标数target。

双指针策略： 对撞指针

图解演示

解题思路：

  1. 左指针指向最小值，右指针指向最大值

  2. 计算两指针指向值的和

  3. 根据和与目标值的关系移动指针

题目2：三数之和

问题描述： 给你一个包含n个整数的数组nums，判断nums中是否存在三个元素a,b,c，使得a+b+c=0？请你找出所有满足条件且不重复的三元组。

双指针策略： 固定一个数，对剩余数组使用对撞指针

图解演示

解题思路：

  1. 先对数组排序

  2. 固定第一个数，对剩余数组使用两数之和的双指针解法

  3. 注意去重：跳过重复的元素

题目3：环形链表检测

问题描述： 给定一个链表，判断链表中是否有环。

双指针策略： 快慢指针（Floyd判圈算法）

图解演示

解题思路：

  1. 慢指针每次移动1步，快指针每次移动2步

  2. 如果有环，快指针最终会追上慢指针

  3. 如果无环，快指针会先到达链表末尾

题目4：删除排序数组中的重复项

问题描述： 给定一个排序数组，你需要在原地删除重复出现的元素，使得每个元素只出现一次。

双指针策略： 快慢指针，慢指针维护结果，快指针探索新元素

图解演示

解题思路：

  1. 慢指针指向当前不重复序列的末尾

  2. 快指针向前探索

  3. 当快指针指向的值与慢指针不同时，更新慢指针位置

题目5：盛最多水的容器

问题描述： 给你n个非负整数a₁,a₂,...,aₙ，每个数代表坐标中的一个点(i,aᵢ)。在坐标内画n条垂直线，垂直线i的两个端点分别为(i,0)和(i,aᵢ)。找出其中的两条线，使得它们与x轴共同构成的容器可以容纳最多的水。

双指针策略： 对撞指针，总是移动较短的边

图解演示

解题思路：

  1. 两个指针分别指向数组两端

  2. 计算当前容器面积并更新最大值

  3. 移动高度较小的指针（移动较大的指针不会得到更大面积）

🔧 双指针设计技巧

指针移动策略

1. 基于比较的移动

比较两指针指向的值 根据比较结果决定移动哪个指针

2. 基于条件的移动

计算当前状态的某个值（如和、差、乘积） 根据这个值与目标的关系移动指针

3. 基于速度的移动

快指针：每次移动k步 慢指针：每次移动1步 用于链表问题

4. 基于贪心的移动

总是移动能够带来最优改进的指针 如盛水容器问题中移动较短边

📈 复杂度分析

时间复杂度

  • 基本双指针 ：O(n)，每个元素最多访问一次

  • 嵌套双指针 ：O(n²)，如三数之和问题

  • 双指针+排序 ：O(n log n)，排序主导复杂度

空间复杂度

  • 纯双指针 ：O(1)，只需要两个指针变量

  • 辅助哈希表 ：O(n)，存储额外信息

  • 递归调用 ：O(log n)到O(n)，取决于递归深度

✅ 适用场景识别

适合双指针的问题特征

  1. 📊 有序数组问题

    • 数组已排序或可以排序

    • 需要查找满足某种关系的元素对

  2. 🔗 链表问题

    • 需要检测环或找到特定位置

    • 需要同时处理不同速度的遍历

  3. 🎯 优化暴力解法

    • 原本需要嵌套循环的问题

    • 可以通过指针移动规律简化

  4. 🔄 区间/窗口问题

    • 需要维护连续区间

    • 区间端点可以根据条件调整

识别关键词

✅ "两个数的和/差/乘积" ✅ "有序数组" ✅ "链表环/中点" ✅ "去重/移动元素" ✅ "最大/最小面积/周长"

不适合双指针的情况

❌ 需要考虑所有可能组合 ❌ 问题没有单调性或规律性 ❌ 数据结构不支持随机访问 ❌ 优化效果不明显

💡 解题模板

对撞指针模板

排序数组（如果需要） left = 0, right = n - 1 while left < right: 计算当前状态值 if 当前值 == 目标值: 处理找到的情况 更新指针（可能需要去重） elif 当前值 < 目标值: left++ // 需要更大的值 else: right-- // 需要更小的值 return 结果

快慢指针模板

slow = head, fast = head while fast != null and fast.next != null: slow = slow.next // 慢指针走1步 fast = fast.next.next // 快指针走2步 if slow == fast: // 检测到环或其他条件 break // 根据问题处理结果 return 结果

滑动窗口指针模板

left = 0, right = 0 初始化窗口状态 while right < n: // 扩展窗口 添加 arr[right] 到窗口 right++ while 窗口不满足条件: // 收缩窗口 移除 arr[left] 从窗口 left++ // 更新答案 更新结果 return 结果
