🔧 重要概念和术语

基础概念

  • Thread（线程） ：程序执行的最小单位

  • Runnable（可运行） ：线程执行的任务接口

  • Synchronized（同步） ：保证线程安全的关键字

  • Volatile（易失性） ：保证可见性的关键字

  • Atomic（原子性） ：不可分割的操作特性

锁相关术语

  • Mutex（互斥锁） ：同时只允许一个线程访问

  • ReentrantLock（可重入锁） ：支持重复获取的锁

  • ReadWriteLock（读写锁） ：读共享、写独占的锁

  • Condition（条件变量） ：线程等待和唤醒机制

  • Fairness（公平性） ：锁获取的公平策略

并发工具术语

  • ThreadPool（线程池） ：线程的集合和管理机制

  • Executor（执行器） ：任务执行框架

  • CountDownLatch（倒计时锁存器） ：等待多个事件完成

  • Semaphore（信号量） ：控制资源访问数量

  • CyclicBarrier（循环屏障） ：多线程同步点

性能相关术语

  • Contention（竞争） ：多线程对资源的争夺

  • Deadlock（死锁） ：线程相互等待造成的阻塞

  • Livelock（活锁） ：线程持续运行但无法前进

  • Starvation（饥饿） ：线程长期无法获得资源

  • Throughput（吞吐量） ：单位时间处理的任务数

🌟 学习重点提示

核心原理理解

  1. Java内存模型 - 理解可见性、有序性、原子性的保证机制

  2. 锁的实现原理 - 从操作系统到JVM的锁实现层次

  3. CAS机制 - 无锁编程的基础，理解其优势和局限性

  4. 线程池原理 - 任务提交、执行、结果获取的完整流程

常见误区避免

  1. 过度同步 - 不必要的同步会影响性能

  2. 锁粒度过大 - 粗粒度锁会降低并发度

  3. 忽略内存可见性 - volatile和synchronized的正确使用

  4. 线程池配置不当 - 参数设置需要根据具体场景调整

📈 技术发展趋势

新特性关注

  • Virtual Threads（虚拟线程） ：Java 19+的轻量级线程
