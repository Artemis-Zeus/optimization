Q2: 如何查看和设置表的存储引擎？

答案：

查看存储引擎：

-- 查看所有支持的存储引擎 SHOW ENGINES; -- 查看特定表的存储引擎 SHOW CREATE TABLE table_name; SHOW TABLE STATUS LIKE 'table_name';

设置存储引擎：

-- 创建表时指定 CREATE TABLE test_table ( id INT PRIMARY KEY, name VARCHAR(50) ) ENGINE=InnoDB; -- 修改现有表的存储引擎 ALTER TABLE test_table ENGINE=MyISAM;

🚀 InnoDB引擎详解

Q3: InnoDB存储引擎有哪些主要特性？

答案：

详细特性：

  1. 事务支持

    • 支持完整的ACID特性

    • 自动提交和回滚机制

    • 支持保存点（Savepoint）

  2. 锁机制

    • 行级锁，支持高并发

    • 间隙锁防止幻读

    • 意向锁提高锁效率

  3. 外键约束

    • 维护表间参照完整性

    • 级联操作支持

    • 自动检查约束

  4. MVCC多版本并发控制

    • 读不阻塞写

    • 版本链管理

    • 快照读实现

Q4: InnoDB的存储结构是怎样的？

答案：

存储层次详解：

  • 表空间（Tablespace） ：存储引擎的最高逻辑层

  • 段（Segment） ：由多个区组成，包括数据段、索引段、回滚段等

  • 区（Extent） ：64个连续的页，固定大小1MB

  • 页（Page） ：InnoDB的最小I/O单位，默认16KB

  • 行（Row） ：实际的数据记录

📊 MyISAM引擎详解

Q5: MyISAM存储引擎有什么特点？适用于什么场景？

答案：

MyISAM优势：

  • 查询性能 ：无事务开销，查询速度快

  • 存储效率 ：支持表压缩，节省存储空间

  • 全文索引 ：内置全文搜索功能

  • 简单维护 ：每个表对应独立文件

MyISAM劣势：

  • 无事务支持 ：不支持事务和外键

  • 表级锁 ：并发写入性能差

  • 崩溃恢复 ：恢复能力较弱

选择建议：

选择InnoDB的场景：

  • 需要事务支持的应用

  • 高并发读写环境

  • 数据一致性要求高

  • 需要外键约束

  • 现代Web应用（推荐）

选择MyISAM的场景：

  • 读多写少的应用

  • 数据仓库和分析系统

  • 需要全文搜索功能

  • 对查询性能要求极高

  • 历史数据存储

🔧 其他存储引擎

Q8: Memory存储引擎有什么特点？适用场景是什么？

答案：

Memory引擎限制：

  • 数据量受内存大小限制

  • 不支持TEXT和BLOB类型

  • 不支持AUTO_INCREMENT

  • 服务器重启数据丢失

Q9: 如何根据业务需求选择合适的存储引擎？

答案：

选择流程图：

决策要素：

  1. 数据一致性需求

    • 高：选择InnoDB

    • 低：可以考虑MyISAM

  2. 并发访问模式

    • 高并发读写：InnoDB

    • 主要是查询：MyISAM

  3. 数据持久性要求

    • 永久存储：InnoDB/MyISAM

    • 临时数据：Memory

  4. 特殊功能需求

    • 全文搜索：MyISAM

    • 外键约束：InnoDB

    • 压缩存储：MyISAM
