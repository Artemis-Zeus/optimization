动态规划详解

🎯 核心思想

动态规划（Dynamic Programming，DP）是一种通过把原问题分解为相对简单的子问题的方式求解复杂问题的方法。它适用于有重叠子问题和最优子结构性质的问题。

基本原理

核心特征：

  • 🔹 最优子结构 ：问题的最优解包含子问题的最优解

  • 🔹 重叠子问题 ：递归过程中会多次求解相同的子问题

  • 🔹 无后效性 ：子问题的解不会因为后续决策而改变

🧠 算法思维模式

DP问题解决步骤

  1. 🎯 定义状态

    • 确定dp数组的含义

    • 选择合适的状态表示

  2. 🔄 找出状态转移方程

    • 分析当前状态如何从之前状态得出

    • 考虑所有可能的转移路径

  3. 🏗️ 确定边界条件

    • 最小规模问题的解

    • 初始化dp数组

  4. 📊 确定计算顺序

    • 保证计算当前状态时依赖的状态已计算

    • 通常从小到大计算

📊 经典题目分析

题目1：斐波那契数列

问题描述： F(0) = 0, F(1) = 1, F(n) = F(n-1) + F(n-2)

状态定义： dp[i] = 第i个斐波那契数 转移方程： dp[i] = dp[i-1] + dp[i-2]

图解演示

解题过程：

  • 重叠子问题：F(3)、F(2)被重复计算

  • DP解法：从F(0)开始，逐步计算到F(n)

  • 时间复杂度：从O(2^n)降到O(n)

题目2：爬楼梯问题

问题描述： 假设你正在爬楼梯。需要n阶你才能到达楼顶。每次你可以爬1或2个台阶。你有多少种不同的方法可以爬到楼顶呢？

状态定义： dp[i] = 爬到第i阶楼梯的方法数 转移方程： dp[i] = dp[i-1] + dp[i-2]

图解演示

核心思路：

  • 要到达第i阶，只能从第(i-1)阶爬1步，或从第(i-2)阶爬2步

  • 因此dp[i] = dp[i-1] + dp[i-2]

题目3：最长递增子序列(LIS)

问题描述： 给定一个无序的整数数组，找到其中最长上升子序列的长度。

状态定义： dp[i] = 以nums[i]结尾的最长递增子序列长度 转移方程： dp[i] = max(dp[j] + 1) for all j < i and nums[j] < nums[i]

图解演示

解题思路：

  1. 对于每个位置i，检查所有之前的位置j

  2. 如果nums[j] < nums[i]，则可以在以nums[j]结尾的LIS后面加上nums[i]

  3. 取所有可能的最大值

题目4：背包问题

问题描述： 有N件物品和一个容量为V的背包。第i件物品的重量是w[i]，价值是v[i]。求解将哪些物品装入背包可使价值总和最大。

状态定义： dp[i][j] = 前i个物品放入容量为j的背包的最大价值 转移方程： dp[i][j] = max(dp[i-1][j], dp[i-1][j-w[i]] + v[i])

图解演示

核心思路：

  • 对于每个物品，有选择和不选择两种决策

  • 选择：当前背包容量减去物品重量，价值加上物品价值

  • 不选：保持之前的最优解

🔧 状态转移方程推导技巧

常见DP类型及其转移方程

1. 线性DP

特征： 状态转移呈线性关系

一维：dp[i] = f(dp[i-1], dp[i-2], ...) 二维：dp[i][j] = f(dp[i-1][j], dp[i][j-1], ...)

2. 区间DP

特征： 按区间长度递推

dp[i][j] = min/max(dp[i][k] + dp[k+1][j] + cost)

3. 树形DP

特征： 在树结构上进行状态转移

dp[node] = f(dp[child1], dp[child2], ...)

4. 状态机DP

特征： 有明确的状态转移图

dp[i][state] = transition from previous states

转移方程推导方法

方法1：递推关系法

  1. 找出问题的递推关系

  2. 确定边界条件

  3. 写出状态转移方程

方法2：决策分析法

  1. 分析每一步的所有可能决策

  2. 对每种决策写出状态转移

  3. 选择最优决策

方法3：阶段分析法

  1. 将问题分解为不同阶段

  2. 分析阶段间的关系

  3. 建立阶段间的转移方程

📈 复杂度分析

时间复杂度

  • 一维DP ：通常O(n)或O(n²)

  • 二维DP ：通常O(n²)或O(nm)

  • 三维DP ：通常O(n³)或O(nmk)

✅ 适用场景识别

动态规划适用的问题特征

  1. 🎯 求最优解问题

    • 最大值/最小值

    • 最长/最短路径

    • 最多/最少数量

  2. 🔄 具有最优子结构

    • 问题的最优解包含子问题的最优解

    • 子问题的最优解可以独立求解

  3. 🌳 具有重叠子问题

    • 递归过程中会重复求解相同子问题

    • 子问题数量多项式级别

  4. 🚫 具有无后效性

    • 当前状态确定后，未来发展不受过去影响

    • 状态转移只依赖当前状态

不适合动态规划的情况

❌ 子问题不重叠 （用分治算法） ❌ 没有最优子结构 （用其他方法） ❌ 状态数量指数级别 （可能需要剪枝或近似算法） ❌ 有后效性问题 （需要其他技巧处理）

💡 解题模板

通用解题框架

1. 问题分析 - 确定这是一个DP问题 - 识别最优子结构和重叠子问题 ↓ 2. 状态设计 - 定义dp数组的含义 - 确定状态维度 ↓ 3. 转移方程 - 分析状态之间的关系 - 写出状态转移方程 ↓ 4. 边界条件 - 确定初始状态 - 处理边界情况 ↓ 5. 计算顺序 - 确定计算的顺序 - 实现算法

数字游戏、计数问题

旅行商问题、任务分配
