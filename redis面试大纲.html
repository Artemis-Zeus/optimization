Redis 技术大纲

一、Redis 基础概念

1.1 Redis 简介

  • 什么是 Redis？

    • Remote Dictionary Server（远程字典服务器）

    • 开源的内存数据结构存储系统

    • 可用作数据库、缓存和消息代理

  • Redis 的特点

    • 基于内存存储，读写速度快

    • 支持多种数据结构

    • 支持数据持久化

    • 支持主从复制

    • 支持集群模式

    • 单线程模型（6.0后支持多线程）

1.2 Redis vs 其他缓存

  • Redis vs Memcached

    • 数据结构：Redis支持多种数据结构，Memcached只支持字符串

    • 持久化：Redis支持持久化，Memcached不支持

    • 集群：Redis原生支持集群，Memcached需要客户端实现

    • 线程模型：Redis单线程，Memcached多线程

  • Redis vs 传统数据库

    • 存储介质：Redis基于内存，传统数据库基于磁盘

    • 数据结构：Redis支持丰富的数据结构

    • 事务支持：Redis支持简单事务，传统数据库支持ACID事务

二、Redis 数据结构

2.1 基本数据类型

String（字符串）

  • 特点 ：最基本的数据类型，二进制安全

  • 应用场景 ：缓存、计数器、分布式锁

  • 常用命令 ：SET、GET、INCR、DECR、APPEND

  • 底层实现 ：SDS（Simple Dynamic String）

Hash（哈希）

  • 特点 ：键值对集合，类似于Java中的HashMap

  • 应用场景 ：存储对象信息

  • 常用命令 ：HSET、HGET、HMSET、HMGET、HGETALL

  • 底层实现 ：ziplist + hashtable

List（列表）

  • 特点 ：有序的字符串列表，支持双端操作

  • 应用场景 ：消息队列、最新消息排行

  • 常用命令 ：LPUSH、RPUSH、LPOP、RPOP、LRANGE

  • 底层实现 ：quicklist（ziplist + linkedlist）

Set（集合）

  • 特点 ：无序的字符串集合，元素唯一

  • 应用场景 ：标签、好友关系、数据去重

  • 常用命令 ：SADD、SREM、SMEMBERS、SINTER、SUNION

  • 底层实现 ：intset + hashtable

Sorted Set（有序集合）

  • 特点 ：有序的字符串集合，每个元素关联一个分数

  • 应用场景 ：排行榜、优先级队列

  • 常用命令 ：ZADD、ZREM、ZRANGE、ZRANK、ZSCORE

  • 底层实现 ：ziplist + skiplist

2.2 高级数据类型

Bitmap（位图）

  • 特点 ：基于字符串类型，按位操作

  • 应用场景 ：用户签到、在线用户统计

  • 常用命令 ：SETBIT、GETBIT、BITCOUNT、BITOP

HyperLogLog

  • 特点 ：用于基数统计的概率数据结构

  • 应用场景 ：大数据量下的唯一值计数

  • 常用命令 ：PFADD、PFCOUNT、PFMERGE

Geospatial（地理位置）

  • 特点 ：存储地理位置信息

  • 应用场景 ：LBS应用、附近的人

  • 常用命令 ：GEOADD、GEODIST、GEORADIUS

Stream（流）

  • 特点 ：Redis 5.0新增，用于消息队列

  • 应用场景 ：消息队列、事件溯源

  • 常用命令 ：XADD、XREAD、XGROUP

三、Redis 持久化

3.1 RDB（Redis Database）

  • 原理 ：将内存中的数据快照保存到磁盘

  • 触发方式：

    • 手动触发：SAVE、BGSAVE

    • 自动触发：配置save规则

  • 优点 ：文件紧凑、恢复速度快、对性能影响小

  • 缺点 ：数据完整性差、fork子进程消耗内存

3.2 AOF（Append Only File）

  • 原理 ：记录每个写操作命令

  • 同步策略：

    • always：每个命令都同步

    • everysec：每秒同步一次

    • no：由操作系统决定

  • 优点 ：数据完整性好、可读性强

  • 缺点 ：文件体积大、恢复速度慢

3.3 混合持久化

  • Redis 4.0新特性 ：RDB + AOF

  • 优点 ：结合两者优势，快速恢复 + 数据完整性

四、Redis 内存管理

4.1 内存淘汰策略

  • noeviction ：不淘汰，内存满时报错

  • allkeys-lru ：所有key中淘汰最近最少使用

  • allkeys-lfu ：所有key中淘汰最少使用频率

  • allkeys-random ：所有key中随机淘汰

  • volatile-lru ：设置过期时间的key中淘汰LRU

  • volatile-lfu ：设置过期时间的key中淘汰LFU

  • volatile-random ：设置过期时间的key中随机淘汰

  • volatile-ttl ：淘汰即将过期的key

4.2 过期策略

  • 定时删除 ：设置定时器，到期立即删除

  • 惰性删除 ：访问时检查是否过期

  • 定期删除 ：定期随机检查并删除过期key

4.3 内存优化

  • 选择合适的数据结构

  • 控制key的长度

  • 使用压缩列表

  • 设置合理的过期时间

五、Redis 事务

5.1 事务特性

  • 原子性 ：事务中的命令要么全部执行，要么全部不执行

  • 一致性 ：事务执行前后数据保持一致

  • 隔离性 ：事务之间相互隔离

  • 持久性 ：事务执行结果会被持久化

5.2 事务命令

  • MULTI ：开启事务

  • EXEC ：执行事务

  • DISCARD ：取消事务

  • WATCH ：监视key，实现乐观锁

5.3 事务的局限性

  • 不支持回滚

  • 命令错误会导致整个事务失败

  • 运行时错误不会影响其他命令执行

六、Redis 集群

6.1 主从复制

  • 原理 ：主节点写，从节点读，数据同步

  • 配置 ：slaveof命令或配置文件

  • 同步过程：

    • 全量同步：首次连接时进行

    • 增量同步：正常运行时进行

  • 优点 ：读写分离、数据备份

  • 缺点 ：主节点故障需要手动切换

6.2 哨兵模式（Sentinel）

  • 作用 ：监控、通知、自动故障转移

  • 工作原理：

    • 监控主从节点状态

    • 主节点故障时自动选举新主节点

    • 通知客户端新的主节点信息

  • 配置 ：sentinel.conf配置文件

6.3 集群模式（Cluster）

  • 特点 ：分布式、高可用、可扩展

  • 数据分片 ：16384个槽位，使用CRC16算法

  • 节点通信 ：Gossip协议

  • 故障转移 ：自动检测和故障转移

  • 优点 ：水平扩展、高可用

  • 缺点 ：复杂度高、不支持多数据库

七、Redis 性能优化

7.1 性能监控

  • INFO命令 ：查看Redis运行状态

  • MONITOR命令 ：实时监控执行的命令

  • SLOWLOG ：慢查询日志

  • Redis性能指标：

    • QPS（每秒查询数）

    • 内存使用率

    • 连接数

    • 命中率

7.2 性能优化策略

  • 合理设计key ：避免过长的key名

  • 选择合适的数据结构

  • 批量操作 ：使用pipeline、mget等

  • 避免大key ：单个key过大影响性能

  • 合理设置过期时间

  • 使用连接池

  • 读写分离

7.3 常见性能问题

  • 慢查询 ：使用SLOWLOG分析

  • 内存碎片 ：重启Redis或使用MEMORY PURGE

  • 大key问题 ：拆分大key或使用scan命令

  • 热点key ：使用本地缓存或分片

八、Redis 应用场景

8.1 缓存

  • 页面缓存 ：缓存整个页面内容

  • 对象缓存 ：缓存数据库查询结果

  • 会话缓存 ：存储用户会话信息

8.2 分布式锁

  • 实现方式 ：SET key value NX EX timeout

  • 问题和解决：

    • 锁超时：设置合理的超时时间

    • 锁误删：使用唯一标识符

    • 锁续期：使用Redisson等工具

8.3 消息队列

  • List实现 ：LPUSH + BRPOP

  • Stream实现 ：XADD + XREAD

  • 发布订阅 ：PUBLISH + SUBSCRIBE

8.4 计数器

  • 网站访问量 ：INCR命令

  • 限流 ：滑动窗口算法

  • 库存扣减 ：原子性操作

8.5 排行榜

  • Sorted Set实现 ：ZADD + ZRANGE

  • 实时更新 ：ZINCRBY命令
