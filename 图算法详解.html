图算法详解

🎯 核心思想

图算法是处理图结构数据的专门算法集合。图由顶点(Vertex)和边(Edge)组成，是一种重要的非线性数据结构，广泛应用于网络、路径规划、社交关系、依赖关系等领域。

基本概念

核心特征：

  • 🔹 顶点集合V ：图中所有节点的集合

  • 🔹 边集合E ：连接顶点的边的集合

  • 🔹 连通性 ：顶点之间是否存在路径

  • 🔹 路径与环 ：顶点序列和回到起点的路径

🧠 算法思维模式

图的表示方法

1. 邻接矩阵

特征： 用二维数组表示图

优点：查询两点是否相邻O(1)，适合稠密图 缺点：空间复杂度O(V²)，不适合稀疏图 存储：matrix[i][j] = 1表示i到j有边

2. 邻接表

特征： 用链表数组表示图

优点：空间复杂度O(V+E)，适合稀疏图 缺点：查询两点是否相邻需要O(度数) 存储：adj[i]存储与顶点i相邻的所有顶点

3. 边集数组

特征： 直接存储所有边的信息

优点：存储简单，适合某些特定算法 缺点：查询效率低 存储：edges[(u,v,weight)]

图算法分类

1. 图遍历算法

  • 深度优先搜索(DFS)

  • 广度优先搜索(BFS)

  • 拓扑排序

2. 最短路径算法

  • Dijkstra算法（单源最短路径）

  • Floyd-Warshall算法（全源最短路径）

  • Bellman-Ford算法（含负权边）

3. 最小生成树算法

  • Kruskal算法

  • Prim算法

4. 强连通分量算法

  • Tarjan算法

  • Kosaraju算法

📊 经典题目分析

题目1：课程表问题（拓扑排序）

问题描述： 你这个学期必须选修numCourses门课程，记为0到numCourses-1。在选修某些课程之前需要一些先修课程。先修课程按数组prerequisites给出，prerequisites[i] = [ai, bi]，表示如果要学习课程ai则必须先学习课程bi。请判断是否可能完成所有课程的学习？

图算法策略： 构建有向图，使用拓扑排序检测环

图解演示

解题思路：

  1. 构建有向图表示课程依赖关系

  2. 计算每个课程的入度

  3. 将入度为0的课程加入队列

  4. 依次处理队列中的课程，更新后续课程的入度

  5. 如果处理的课程数等于总课程数，则可以完成

题目2：网络延迟时间（Dijkstra算法）

问题描述： 有n个网络节点，标记为1到n。给你一个列表times，表示信号经过有向边的传递时间。现在，从某个节点K发出一个信号，需要多久才能使所有节点都收到信号？如果不能使所有节点收到信号，返回-1。

图算法策略： 单源最短路径问题，使用Dijkstra算法

图解演示

解题思路：

  1. 构建加权有向图

  2. 使用优先队列实现Dijkstra算法

  3. 从源点开始，每次选择距离最小的未访问节点

  4. 更新该节点所有邻居的最短距离

  5. 重复直到所有节点都被访问

  6. 返回最大的最短距离

题目3：省份数量（并查集/DFS）

问题描述： 有n个城市，其中一些彼此相连，另一些没有相连。如果城市a与城市b直接相连，且城市b与城市c直接相连，那么城市a与城市c间接相连。省份是一组直接或间接相连的城市，组内不含其他没有相连的城市。给你一个n×n的矩阵isConnected，其中isConnected[i][j] = 1表示第i个城市和第j个城市直接相连，否则为0。返回矩阵中省份的数量。

图算法策略： 连通分量问题，可用DFS或并查集

图解演示

解题思路（DFS）：

  1. 将邻接矩阵看作无向图

  2. 对每个未访问的城市进行DFS

  3. 每次DFS遍历一个连通分量（省份）

  4. 统计DFS的次数即为省份数量

题目4：最小生成树（Kruskal算法）

问题描述： 想要连通所有城市，每条边都有一定的成本。找到连通所有城市的最小成本。

图算法策略： 最小生成树问题，使用Kruskal或Prim算法

图解演示

解题思路（Kruskal）：

  1. 将所有边按权重从小到大排序

  2. 使用并查集维护连通分量

  3. 依次考虑每条边，如果不形成环则加入生成树

  4. 直到生成树有n-1条边

🔧 图算法设计技巧

图的存储优化

1. 稠密图vs稀疏图

稠密图(边数接近V²): 使用邻接矩阵 稀疏图(边数远小于V²): 使用邻接表

2. 带权图的存储

邻接矩阵: matrix[i][j] = weight 邻接表: adj[i].append((j, weight)) 边集数组: edges.append((u, v, weight))

💡 解题模板

DFS遍历模板

def dfs(graph, start, visited): visited.add(start) process(start) # 处理当前节点 for neighbor in graph[start]: if neighbor not in visited: dfs(graph, neighbor, visited)

BFS遍历模板

from collections import deque def bfs(graph, start): queue = deque([start]) visited = {start} while queue: node = queue.popleft() process(node) # 处理当前节点 for neighbor in graph[node]: if neighbor not in visited: visited.add(neighbor) queue.append(neighbor)

Dijkstra算法模板

import heapq def dijkstra(graph, start): distances = {node: float('inf') for node in graph} distances[start] = 0 pq = [(0, start)] while pq: current_distance, current = heapq.heappop(pq) if current_distance > distances[current]: continue for neighbor, weight in graph[current]: distance = current_distance + weight if distance < distances[neighbor]: distances[neighbor] = distance heapq.heappush(pq, (distance, neighbor)) return distances

拓扑排序模板

from collections import deque, defaultdict def topological_sort(graph, n): in_degree = [0] * n adj = defaultdict(list) # 构建图和计算入度 for u, v in graph: adj[u].append(v) in_degree[v] += 1 # 找到所有入度为0的节点 queue = deque([i for i in range(n) if in_degree[i] == 0]) result = [] while queue: node = queue.popleft() result.append(node) for neighbor in adj[node]: in_degree[neighbor] -= 1 if in_degree[neighbor] == 0: queue.append(neighbor) return result if len(result) == n else []

🔄 图算法优化技巧

1. 路径压缩（并查集）

在查找过程中压缩路径，提高后续查找效率

2. 启发式搜索（A*）

使用启发式函数指导搜索方向，提高搜索效率

3. 双向搜索

从起点和终点同时搜索，在中间相遇

4. 记忆化搜索

缓存中间结果，避免重复计算

💡 记忆要点 ：图算法的关键是选择合适的图表示方法和对应的算法。理解DFS和BFS是基础，掌握最短路径和最小生成树算法是进阶。注意图的特性（有向/无向、有权/无权）决定算法选择。
